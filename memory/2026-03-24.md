## 核心教训：规则修改完整流程

### 问题的根源
- 规则存在但没被触发 → 不是规则的问题，是创建时没站在全局思考
- 补漏洞式的修改 → 从局部出发，不是全局出发
- 流程缺少环节 → 审查规则没被触发

### 完整流程（铁律）
1. 前置思考：站在全局想清楚，对全局有什么影响？有没有副作用？
2. 类型判断：是修改还是创建？
3. 执行修改
4. 审查闭环：确认审查规则是否被触发、是否通过
5. 更新索引

### 全局思考优先原则
- 遇到规则修改/创建，先站在全局想清楚
- 不是修补一个点，是看这条改动对全局的影响
- 这个前置思考，不是可选项，是所有规则操作的必经步骤

### 区别于"前置三问"
- 前置三问：审查规则写入前要过的问题（解决什么/不改行吗/影响全局吗）
- 全局思考优先：所有规则操作的前置思考（不是审查，是思考）
- 两个是不同场景，不能混淆

## 缠论中枢算法关键教训

### 核心理解（纠正多次错误后）
- **三笔同向**是构成中枢的必要条件（不是"任意三笔"）
- 三笔同向但无重叠区间 = 没有中枢 = 没有背驰
- 有线段（连续3+个同向笔）= 趋势发展中 = 无中枢 = 观察为主
- 无结构 = 不进场，这是缠论的纪律

### 正确判断
- 润建股份7笔：无三笔连续同向 → 无中枢 → 评分4.0/10 → 观望（正确）
- 德业股份5笔：无三笔连续同向 → 无中枢 → RSI超买 → 谨慎（正确）

### 代码bug记录
- bug1：重叠判断条件写反（ZG>ZD应为ZG<ZD）
- bug2：三笔中枢公式缠论表述不清 → 理解"三笔同向且重叠"
- bug3：要求"三笔"但数据不足120天笔数不够

### 缠论核心口诀（自验证）
- 有线段 = 趋势发展中 = 没有中枢 = 等中枢形成才操作
- 没有结构 = 不进场 = 这是纪律不是预测
