## 核心流程图

### 规则修改完整流程（铁律）
```
① 前置思考（全局）
   → 这个改动对全局有什么影响？
   → 会不会影响其他规则？
   → 会不会有副作用？
② 类型判断
   → 是修改还是创建？
③ 执行修改
④ 审查闭环
   → 审查规则是否被触发？
   → 是否通过？
⑤ 更新索引
   → rules-index.md
```
⚠️ 五步缺一不可，跳过任意一步则规则修改无效

### 系统触发链
```
Session开始
  → system-rules.md 启动
  → session-rules.md 加载
  → 今日记忆加载

任务执行
  → execution-standards 执行
  → 执行感知 监控
  → 执行感知 复盘

规则修改/创建
  → rule-creation 触发
  → 审查规则 触发
  → 备份规则 触发
  → 更新索引

定期自检
  → rule-execution-checklist 自检
  → 发现问题 → 规则修改流程
```

### 全局自检固定项（每次自检必问）
1. 有哪些规则我知道但没用？
2. 执行了哪些执行有偏差的规则？
3. 规则之间有没有冲突？
4. 规则有没有被"框住"不敢打破的情况？
