# 文件管理规则

## 🎯 规则目的
规范文件操作，确保文件按照既定的目录结构进行管理，避免混乱和错误。

## 🎯 文件分类规则

### 1. 学习资料类 → knowledge/ 目录
```
✅ 保存内容：
- 技术文档（如：LocalAI命令参数文档.md）
- 项目分析报告（如：LocalAI项目分析报告.md）
- 学习笔记和教程
- 参考资料和指南

❌ 不保存内容：
- 每日工作记录
- 系统配置信息
- 临时操作日志
```

### 2. 记忆系统类 → memory/ 目录
```
✅ 保存内容：
- YYYY-MM-DD.md（每日记忆）
- rules-index.md（规则索引）
- rules/（规则文件）

❌ 不保存内容：
- 学习资料
- 技术文档
- 项目文件
```

### 3. 项目管理类 → projects/ 目录
```
✅ 保存内容：
- 项目代码
- 项目文档
- 项目配置
- 项目日志

❌ 不保存内容：
- 系统配置
- 学习资料
```

### 4. 工具配置类 → tools/ 目录
```
✅ 保存内容：
- 外部工具配置
- 插件设置

❌ 不保存内容：
- 项目文件
- 学习资料
```

## 🚨 文件操作触发规则

### 触发条件
当用户提出以下请求时，必须触发文件管理规则：

1. **保存文档请求**
   - 用户说："保存这个文档"
   - 用户说："把这个文件保存到..."
   - 用户说："记录一下这个信息"

2. **文件类型判断**
   - 学习资料 → knowledge/
   - 系统记忆 → memory/
   - 项目文件 → projects/
   - 工具配置 → tools/

3. **文件命名规则**
   - 使用中文文件名
   - 文件名要清晰表达内容
   - 避免使用特殊字符和空格
   - 使用.md格式（文档类）

### 操作流程
```
用户请求文件保存
    ↓
触发文件管理规则
    ↓
判断文件类型和分类
    ↓
选择正确的目录
    ↓
检查目录是否存在
    ↓
[如果不存在] 自动创建目录
    ↓
验证目录创建结果
    ↓
应用文件命名规则
    ↓
执行保存操作
    ↓
记录操作到记忆系统
```

## 📋 文件命名规范

### 中文文件命名规则
```
✅ 正确示例：
- LocalAI命令参数文档.md
- iFlow网关部署指南.md
- OpenClaw工作记录.md
- Git使用教程.md

❌ 错误示例：
- localai-chinese-docs.md
- analysis-report-20260318.md
- file1.txt
- document.pdf
```

### 文件名原则
1. **清晰明了**：一看就知道是什么内容
2. **语言统一**：全部使用中文
3. **格式规范**：使用.md后缀
4. **避免冲突**：不要重复命名

## 🔧 文件操作检查清单

### 保存文件前检查
- [ ] 是否触发了文件管理规则？
- [ ] 文件类型判断是否正确？
- [ ] 目录选择是否合适？
- [ ] 目录是否存在？（不存在则自动创建）
- [ ] 目录创建是否成功？
- [ ] 文件命名是否符合规范？
- [ ] 是否会占用记忆系统空间？

### 常见错误检查
- [ ] 避免将学习资料放入memory/
- [ ] 避免将系统配置放入knowledge/
- [ ] 避免使用英文文件名
- [ ] 避免使用特殊字符

## 🎯 特殊情况处理

### 用户指定位置
如果用户明确指定了保存位置，优先遵循用户指示，但仍需检查：
- 路径是否存在
- 权限是否足够
- 是否符合文件分类原则

### 文件冲突处理
- 如果文件已存在，询问用户是否覆盖
- 如果是版本控制文件，建议使用版本管理
- 如果是配置文件，建议备份原文件

## 🔧 目录自动创建规则

### 自动创建触发条件
当以下情况发生时，自动创建目标目录：
1. **文件保存操作**：用户要求保存文件到指定目录
2. **目录不存在**：目标目录在文件系统中不存在
3. **权限允许**：当前用户有创建目录的权限
4. **路径有效**：目标路径格式正确

### 目录创建逻辑
```
目标目录路径解析
    ↓
检查目录是否存在
    ↓
[存在] 继续执行保存操作
    ↓
[不存在] 创建父目录（递归）
    ↓
[创建成功] 继续执行保存操作
    ↓
[创建失败] 报告错误并请求用户干预
```

### 目录创建优先级
```
1. knowledge/ - 学习资料目录（高优先级）
2. memory/   - 记忆系统目录（高优先级）
3. projects/ - 项目管理目录（中优先级）
4. tools/    - 工具配置目录（中优先级）
5. scripts/  - 脚本执行目录（中优先级）
6. sessions/ - 会话管理目录（低优先级）
```

### 创建失败处理
如果目录创建失败，系统应该：
1. **报告错误**：明确说明创建失败的原因
2. **请求权限**：询问是否需要提升权限
3. **提供替代方案**：建议其他保存位置
4. **记录日志**：在记忆系统中记录失败事件

### 目录权限验证
在创建目录前，系统应该验证：
1. **路径权限**：检查当前用户是否有写入权限
2. **磁盘空间**：检查是否有足够的磁盘空间
3. **路径有效性**：验证路径格式是否正确
4. **安全性**：检查路径是否包含危险字符

## 📊 文件管理优先级
