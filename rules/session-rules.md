# Session管理规则

## 核心逻辑

每次开新session前，目的要清楚，不能盲目开始。

## 完整闭环流程

### 第一步：Session结束前（保存）
- **执行中顺手写**：形成结论/学到东西/分析完股票→立刻写进对应记忆文件（不等待，不过夜）
- **待办同步**：待做的事→写入 task-center.md，不只留在记忆里
- 知识 → 归档到对应记忆文件
- 待办 → 写入 task-center.md
- 任务涉及新文件 → 归入对应目录 + 更新该目录SUMMARY.md
- **触发规则：写完就触发审查（是否完整闭环？）**

### 第二步：新Session开始前（准备）
- 检查记忆文件是否完整
- **写入要加载的内容**（这次要做什么）
- **触发规则：检查是否有待处理事项**

### 第三步：新Session开始时（加载）
- 读取MEMORY.md（总索引）→ 了解各目录SUMMARY位置
- 读取memory/SUMMARY.md（记忆索引）→ 今日记忆位置
- 读取memory/YYYY-MM-DD.md（今日记忆）
- **读取tasks/task-center.md**（任务中心）→ 有未完成任务立刻执行
- **自我记忆完整性检查**：确认以为自己记得的东西是否真的还记得
- 加载完成后 → **删除加载记录**
- 无待办 → 正常进入新Session

### 执行触发（铁律）
- 毅哥给出明确任务 → **立刻执行，不需要确认方向，不需要问下一步**
- 遇到障碍 → 先尝试解决，解决不了才问
- 规则和执行冲突时 → 规则优先，但规则本身有漏洞时先补规则再执行

### 第四步：执行
- 做该做的事

### 第五步：完成
- 任务归档到tasks/done/（任务完成的勋章）
- 任务涉及新文件 → 归入对应目录 + 更新该目录SUMMARY.md
- 闭环翻篇

### 异常处理
- 第二步检查时发现有待处理事项 → 立刻加载并执行
- 加载后发现还有其他待办 → 按优先级排序执行
- 无待办 → 正常进入新session

## 触发规则

任何规则被写入或修改 → 自动触发审查规则（rules/审查规则.md）→ 保证所有规则完美闭环。

**没触发就不对。触发是规则的一部分，不是可选项。**

## 本质

**加载即删除，闭环才能翻篇。**

要加载的内容是在开始前就设定好的，不是开始后才定的。
