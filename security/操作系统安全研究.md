## 一、我们的威胁模型

OpenClaw作为一个AI助手系统，面临以下安全威胁：

### 1.1 攻击面分析

```
外部攻击者
    ├── 网络层：恶意连接、端口扫描、中间人攻击
    ├── API层：注入恶意指令、污染上下文
    └── 社会工程：诱导执行危险操作

内部威胁
    ├── 子Agent失控：越权操作、异常行为
    ├── 恶意代码执行：workspace文件被污染
    └── 权限滥用：过度的exec/文件操作

系统层
    ├── 依赖漏洞：npm/node/python包的已知漏洞
    ├── 配置错误：端口暴露、密钥泄露
    └── 权限过宽：过度的系统访问
```

### 1.2 核心资产

- **OpenClaw配置文件**（规则、记忆、密钥）
- **workspace文件**（项目代码、数据、策略）
- **网络通信**（webhook、API调用、大模型通信）
- **执行环境**（exec命令、文件读写、进程控制）
- **子Agent会话**（小智等）

## 二、macOS安全架构

> OpenClaw运行环境（Kevin的iMac，Darwin 21.6.0）

### 2.1 安全机制层次

```
硬件层（Apple Silicon / T2）
├── Secure Enclave：密钥存储、加密运算
├── 硬件级安全启动：防止启动时注入恶意代码
└── 设备完整性验证

引导层（Boot ROM → iBoot → macOS）
├── 链式验证：每步验证下一步的完整性
├── SIP保护：引导过程不可篡改
└── 安全启动模式：只运行签名的Apple代码

系统层（macOS内置安全）
├── SIP（System Integrity Protection）
├── Gatekeeper + Notarization
├── XProtect（恶意软件检测）
├── App Sandbox
├── Hardened Runtime
└── TCC（Transparency, Consent, Control）
```

### 2.2 关键安全机制详解

#### SIP（System Integrity Protection）

**作用**：限制root用户权限，防止系统关键目录被修改

**受保护目录**：
```
/System
/usr（部分）
/sbin
/bin
/var（部分）
```

**对我们的影响**：
- OpenClaw安装在`~/.openclaw`，不在SIP保护范围内
- 需要额外加固workspace目录

**风险**：如果攻击者拿到root权限，可以：
- 修改OpenClaw规则文件（注入恶意规则）
- 替换可执行文件（植入后门）
- 访问内存/会话数据

#### Gatekeeper + Notarization

**作用**：确保只有来自可信来源的App才能运行

**对我们的影响**：
- 第三方工具（如FFmpeg、curl）不受此保护
- 自研脚本需要手动允许运行（`xattr -d com.apple.quarantine`）

#### App Sandbox

**作用**：限制App只能访问明确授权的资源

**对我们的影响**：
- OpenClaw默认不运行在Sandbox中
- 如果要隔离，需要自行实现类似沙箱
- exec工具是主要攻击向量

#### Hardened Runtime

**作用**：防止代码注入、内存篡改、DYLD注入

**对我们的意义**：
- 保护OpenClaw进程不被恶意库注入
- 启用方式：`codesign -f -v -s OpenClaw Hardened Runtime`

#### TCC（Transparency, Consent, Control）

**作用**：管理隐私权限（相机、麦克风、文件、位置）

**对我们的风险**：
- 如果恶意软件获取了TCC权限，可以：
  - 访问文件系统（Desktop、Documents等）
  - 监控输入设备
  - 获取屏幕录制权限

### 2.3 macOS网络层安全

**内置防火墙选项**：
- `pfctl`：BSD packet filter，功能强大但配置复杂
- `ipfw`：已废弃，macOS 10.12后被pf取代
- Application Layer Firewall（ALF）：应用级防火墙，macOS自带
- `netstat` / `lsof`：网络连接监控

**现有工具**：
- `curl` → 可被监控进出的HTTP/HTTPS请求
- `nc` / `netcat` → 网络测试工具
- SSH → 已配置密钥认证

### 2.4 macOS已知攻击向量

| 攻击类型 | 描述 | 严重程度 | 防御方案 |
|----------|------|----------|----------|
| 恶意DMG | 伪装成合法软件的安装包 | 高 | Gatekeeper |
| 提权漏洞 | 本地用户→root | 高 | 系统更新 |
| XPC滥用 | 跨进程通信被劫持 | 中 | 代码签名 |
| TCC欺骗 | 诱导用户授权 | 中 | 用户教育 |
| 恶意输入法 | 监控键盘输入 | 高 | 第三方输入法审查 |
| DYLD注入 | 库劫持 | 高 | Hardened Runtime |
| 恶意LaunchAgent | 开机自启后门 | 高 | 检查~/Library/LaunchAgents |

## 三、Linux安全架构

> 小智运行环境（Debian，192.168.8.171）

### 3.1 安全机制层次

```
内核层（Kernel）
├── SELinux / AppArmor（强制访问控制）
├── seccomp（系统调用过滤）
├── namespaces（资源隔离）
├── cgroups（资源限制）
└──内核加固（ASLR、DEP、KPTI）

系统层
├── UFW / iptables / nftables（netfilter防火墙）
├── fail2ban（入侵防御）
├── PAM（认证模块）
└── Auditd（审计日志）
```

### 3.2 关键安全机制

#### iptables / nftables

**作用**：Linux内核级包过滤

**基础链**：
```
INPUT → 入站数据包
OUTPUT → 出站数据包  
FORWARD → 路由转发
```

**示例规则**：
```bash
# 允许已建立的连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许SSH（仅特定IP）
iptables -A INPUT -p tcp -s 192.168.8.0/24 --dport 22 -j ACCEPT

# 丢弃所有其他入站
iptables -A INPUT -j DROP
```

**对我们的意义**：
- 可以限制小智的网络访问范围
- 只允许必要的API调用（iflow-cli等）

#### AppArmor / SELinux

**AppArmor（Debian默认）**：
- 基于路径的访问控制
- 比SELinux更简单
- 配置位于`/etc/apparmor.d/`

**我们的风险**：
- iflow-cli等外部进程可能需要配置文件
- 恶意文件被拉入后可能逃逸

#### Seccomp

**作用**：过滤进程可执行的系统调用

**典型配置**：
```json
{
  "defaultAction": "SCMP_ACT_KILL",
  "syscalls": [
    {"name": "read"},
    {"name": "write"},
    {"name": "exit"}
  ]
}
```

**对我们的意义**：
- 限制exec调用的子进程系统调用
- 防止恶意程序执行危险操作

#### Namespaces & Cgroups

**作用**：容器隔离的基础

**类型**：
- PID namespace：进程隔离
- Network namespace：网络隔离
- Mount namespace：文件系统隔离
- User namespace：用户隔离

**我们的应用**：
- 小智运行在独立namespace中
- 限制其对主机资源的访问

### 3.3 Linux已知风险

| 风险类型 | 描述 | 防御 |
|----------|------|------|
| 提权漏洞 | 内核或SUID程序漏洞 | 及时更新 |
| 服务暴露 | SSH/服务端口对外 | 防火墙+IP限制 |
| 恶意Cron | 定时任务后门 | 检查cron.d |
| 供应链攻击 | npm/pip包被篡改 | 校验hash |
| 容器逃逸 | Docker/LXC漏洞 | 最小权限+隔离 |
| SSH暴力破解 | 弱密码被猜解 | fail2ban+密钥 |

## 四.5 Android安全架构

> 全球最大移动操作系统，基于Linux内核

### 4.5.1 架构层次

```
应用层（APK）
    ↓
Android Framework（Java API）
    ↓
Native库（OpenGL, SQLite, WebKit, SSL）
    ↓
Android Runtime（ART虚拟机）
    ↓
Linux内核（修改版）
```

### 4.5.2 核心安全机制

**应用沙箱：**
- 每个App独立进程 + 独立UID
- 数据隔离：只有App自己和root能访问私有数据
- 权限系统：运行时权限授予（Android 6.0+）

**签名机制：**
- 所有APK必须使用开发者证书签名
- Google Play Protect自动检测篡改应用
- 签名验证防降级攻击

**SELinux：**
- Android默认启用SELinux（ enforcing模式）
- 限制系统服务和App的权限

**Keystore / KeyChain：**
- 硬件级密钥存储（TEE/StrongBox）
- 生物认证绑定

**加密：**
- File-Based Encryption（FBE）：Android 7.0+
- Full-Disk Encryption（FDE）：Android 5.0+
- Adiantum：低性能设备全盘加密

**生物认证：**
- 指纹、面部识别
- 与Keystore深度集成

### 4.5.3 主要攻击向量

| 风险 | 描述 | 防御 |
|------|------|------|
| APK重打包 | 篡改后重签名上架 | Play Protect + 代码混淆 |
| 权限滥用 | App过度申请权限 | 运行时权限+权限审计 |
| 内核漏洞 | Linux内核漏洞提权 | 及时安全更新 |
| 供应链攻击 | SDK/依赖库被污染 | Play库审核 |
| 侧信道攻击 | 锁屏绕过、指纹欺骗 | 硬件绑定+行为检测 |

- 全栈加密：存储、传输、使用全程加密

**应用安全：**
- 签名验证：所有HAP必须签名

### 7.2 网络防火墙（nwfw）实现

```python
# nwfw.py - 网络防火墙核心
# 依赖：lsof, netstat, ps

ALLOWED_DOMAINS = {
    "minimax-portal.com",    # 大模型API
    "openai.com",           # OpenAI（备用）
    "api.openclaw.ai",      # OpenClaw官方
    "eastmoney.com",        # 股票数据
    "sina.com.cn",         # 新浪财经
    "iflow.ai",            # iflow大模型
    "github.com",          # GitHub
}

ALLOWED_PORTS = {443, 80, 22, 2288}  # 允许端口

class NetworkFirewall:
    def __init__(self):
        self.connection_log = []
        self.alert_threshold = 50  # 5分钟内超过50次连接=告警
        
    def scan_connections(self):
        """扫描当前所有网络连接"""
        result = subprocess.run(
            ["lsof", "-i", "-n", "-P"],
            capture_output=True, text=True
        )
        connections = []
        for line in result.stdout.split("\n")[1:]:
            parts = line.split()
            if len(parts) >= 9:
                connections.append({
                    "process": parts[0],
                    "protocol": parts[7],
                    "address": parts[8]
                })
        return connections
    
    def check_connection(self, conn):
        """检查单个连接是否合规"""
        addr = conn["address"]
        # 解析目标地址
        if "->" in addr:
            target = addr.split("->")[1].split(":")[0]
            port = int(addr.split(":")[-1])
        else:
            target = addr.split(":")[0]
            port = int(addr.split(":")[-1]) if ":" in addr else 0
        
        # 检查端口白名单
        if port not in ALLOWED_PORTS:
            return False, f"禁止端口: {port}"
        
        # 检查域名白名单（简化版）
        for domain in ALLOWED_DOMAINS:
            if domain in target:
                return True, "允许"
        
        return False, f"未授权目标: {target}"
    
    def run(self):
        """主循环：每10秒检查一次"""
        while True:
            conns = self.scan_connections()
            for conn in conns:
                allowed, reason = self.check_connection(conn)
                if not allowed:
                    self.alert(conn, reason)
            time.sleep(10)
    
    def alert(self, conn, reason):
        """告警：可疑连接"""
        msg = f"🚨 网络告警: {conn['process']} -> {conn['address']} | 原因: {reason}"
        print(msg)
        # 告警机制：写入日志+推送到webchat
```

### 7.3 进程防火墙（pfw）实现

```python
# pfw.py - 进程防火墙核心
# 目标：限制exec可执行的命令，防止恶意注入

EXEC_WHITELIST = {
    # 系统基础
    "python3", "python", "node", "npm",
    # 网络工具
    "curl", "wget", "ssh", "scp", "rsync",
    # 文件工具
    "ls", "cat", "grep", "awk", "sed", "find",
    # Git
    "git",
    # 自有脚本
    "/Users/kevin/.openclaw/workspace/stock/monitor.py",
    "/Users/kevin/.openclaw/workspace/stock/quant_system.py",
}

DANGEROUS_COMMANDS = {
    "rm -rf", "mkfs", "dd", ":(){:|:&};:",  # 危险命令模式
    "curl | sh", "wget | sh",                  # 管道执行
    "> /dev/sd", "cat /dev/",                  # 磁盘操作
}

class ProcessFirewall:
    def __init__(self):
        self.audit_log = []
    
    def validate_command(self, cmd):
        """验证命令是否在白名单或包含危险模式"""
        import shlex
        tokens = shlex.split(cmd)
        base = tokens[0].split("/")[-1]  # 命令名
        
        # 检查白名单
        if base in EXEC_WHITELIST or cmd.startswith("/"):
            # 检查危险模式
            for danger in DANGEROUS_COMMANDS:
                if danger in cmd:
                    return False, f"危险模式: {danger}"
            return True, "允许"
        
        # 检查是否包含路径（在白名单路径列表中）
        for allowed in EXEC_WHITELIST:
            if allowed.startswith("/") and cmd.startswith(allowed):
                return True, "允许"
        
        return False, f"未授权命令: {base}"
    
    def log_execution(self, cmd, result, allowed):
        """记录执行日志"""
        entry = {
            "time": datetime.now().isoformat(),
            "cmd": cmd,
            "allowed": allowed,
            "result": "executed" if allowed else "blocked"
        }
        self.audit_log.append(entry)
        if not allowed:
            print(f"🚨 已阻止危险命令: {cmd}")
```

### 7.4 文件防火墙（ffw）实现

```python
# ffw.py - 文件完整性监控
# 目标：监控workspace/规则文件的完整性

import hashlib, json, time

PROTECTED_PATHS = [
    "/Users/kevin/.openclaw/workspace/rules/",
    "/Users/kevin/.openclaw/workspace/memory/",
    "/Users/kevin/.openclaw/workspace/SOUL.md",
    "/Users/kevin/.openclaw/workspace/MEMORY.md",
    "/Users/kevin/.openclaw/workspace/AGENTS.md",
]

CRITICAL_PATTERNS = [
    "eval(", "exec(", "os.system", "subprocess",
    "import os", "import sys", "requests.",
    "urllib", "socket",
]

class FileFirewall:
    def __init__(self):
        self.baseline = {}
        self.build_baseline()
    
    def sha256(self, filepath):
        h = hashlib.sha256()
        with open(filepath, "rb") as f:
            for chunk in iter(lambda: f.read(8192), b""):
                h.update(chunk)
        return h.hexdigest()
    
    def build_baseline(self):
        """建立基线：记录所有受保护文件的hash"""
        for path in PROTECTED_PATHS:
            if os.path.isfile(path):
                self.baseline[path] = self.sha256(path)
            elif os.path.isdir(path):
                for root, dirs, files in os.walk(path):
                    for f in files:
                        fp = os.path.join(root, f)
                        self.baseline[fp] = self.sha256(fp)
        
        # 保存基线
        with open("/tmp/ffw_baseline.json", "w") as f:
            json.dump(self.baseline, f, indent=2)
    
    def check_integrity(self):
        """检查文件完整性"""
        changes = []
        for path, expected_hash in self.baseline.items():
            if not os.path.exists(path):
                changes.append({"path": path, "type": "DELETED"})
                continue
            
            current_hash = self.sha256(path)
            if current_hash != expected_hash:
                # 检查是否包含可疑代码
                suspicious = self.scan_content(path)
                changes.append({
                    "path": path,
                    "type": "MODIFIED",
                    "suspicious": suspicious
                })
        
        # 检查新增文件
        for root, dirs, files in os.walk("/Users/kevin/.openclaw/workspace"):
            for f in files:
                fp = os.path.join(root, f)
                if fp not in self.baseline:
                    suspicious = self.scan_content(fp)
                    if suspicious:
                        changes.append({"path": fp, "type": "NEW_SUSPICIOUS", "content": suspicious})
        
        return changes
    
    def scan_content(self, filepath):
        """扫描文件是否包含可疑内容"""
        suspicious = []
        try:
            with open(filepath, "r", errors="ignore") as f:
                content = f.read()
            for pattern in CRITICAL_PATTERNS:
                if pattern in content:
                    suspicious.append(pattern)
        except:
            pass
        return suspicious
    
    def run(self):
        """每60秒检查一次"""
        while True:
            changes = self.check_integrity()
            if changes:
                print(f"🚨 检测到文件变更: {len(changes)}处")
                for c in changes:
                    print(f"  {c['type']}: {c['path']}")
            time.sleep(60)
```
